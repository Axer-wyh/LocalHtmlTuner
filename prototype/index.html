四个高频调优场景

A

局部视觉调优

点选按钮、标题、卡片或表单，让 AI 只处理目标附近样式。

B

区域布局调优

框选一整块布局，携带截图、DOM、尺寸和共同父级上下文。

C

局部交互调优

把卡片展开、hover、点击反馈写进真实 HTML/CSS/JS。

D

快速回退

每次修改前先创建快照，不满意就撤销或恢复历史版本。

从选择到回档的闭环

绑定

连接 companion 服务，选择本地目录或拖入 HTML 文件。

选择

在 `localhost` 预览页点选元素或框选区域。

执行

本地服务创建快照，调用已配置的 CLI 或 API 模型。

确认

刷新预览，查看前后截图，继续微调或撤销。

CONTEXT PACKET 
{
  "mode": "region",
  "target": "section.card-grid",
  "files": ["index.html", "styles.css"],
  "request": "调整间距和层级"
}

浏览器负责看见，本地服务负责修改。

插件不直接写文件。目录管理、快照、CLI 检测、补丁应用和回档都由本地 companion 服务完成。

  • 选区上下文包含 DOM 路径、截图引用、尺寸和计算样式。
  • AI 工具缺失或测试失败时，修改任务不会提交。
  • API Key 仅在用户选择 API / BYOK 模式时由本地保存。

安全边界先于自动化

默认修改范围被限制在绑定项目目录内。普通日志不输出 token、密钥或 `.env` 值。
