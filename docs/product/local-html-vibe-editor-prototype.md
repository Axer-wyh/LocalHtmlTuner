# 本地 HTML Vibe Coding 调优插件原型文档

## 1. 原型目标

本文档描述浏览器插件 MVP 的页面结构、内容、交互和动效。原型服务于静态 HTML/CSS/JS 项目的局部调优流程：用户通过插件绑定本地项目，在 `localhost` 预览页中点选元素或框选区域，在 AI 会话区输入修改要求，由本地 companion 服务调用已配置的本机 AI 工具 CLI 或 API 模型修改本地代码，并支持预览、对比、撤销和回档。

## 2. 产品入口与信息架构

### 2.1 入口形态

- 浏览器工具栏插件入口：点击后默认在当前浏览器页面打开右侧边栏面板。
- 页面右侧边栏面板：插件开启后的默认工作形态，承载 AI 会话工作台。
- 独立小窗口：用户可以点击右上角切换图标，将插件从浏览器页面右侧边栏切换为独立于浏览器内容区的小窗口。
- 页面注入选择层：用户开启点选或框选后，插件在当前网页上注入蒙版、高亮框、浮动提示和框选控件。

### 2.2 插件页面地图

```text
插件入口
  -> 连接检查页
  -> 首次配置页
    -> 本地服务连接
    -> 本机 AI CLI 检测
    -> CLI / API 模式配置
    -> 项目绑定
  -> AI 会话工作台
    -> 项目信息与操作栏
    -> AI 会话区
    -> 底部需求输入框
    -> 修改结果卡片
  -> 对比与回档
    -> 前后对比
    -> 快照历史
    -> 撤销/恢复
  -> 设置
    -> AI 工具配置
    -> 本地服务
    -> GitHub 增强
    -> 安全与日志
```

## 3. 全局布局与状态

### 3.1 默认右侧边栏面板

插件开启后默认与当前浏览器页面绑定，并在页面右侧打开固定边栏面板。

- 建议宽度：380-420px，高度占满当前浏览器视口。
- 顶部显示项目名、预览地址、AI 工具状态、设置入口和窗口切换图标。
- 中部默认是 AI 会话区，而不是表单式任务页。
- 底部固定需求输入框，用户始终可以向 AI 发出具体修改要求。
- 面板可收起，收起后保留一个窄浮动按钮。

### 3.2 独立小窗口

用户点击面板右上角的切换图标后，插件从右侧边栏切换为独立小窗口。该形态适合用户需要完整查看页面右侧内容，或希望把 AI 会话区放在浏览器外侧时使用。

- 建议尺寸：宽 420px，高 640px，可拖动和调整大小。
- 小窗口保留与右侧边栏相同的项目状态、AI 会话和输入框。
- 右上角切换图标可将小窗口恢复为页面右侧边栏。
- 切换形态时保留当前会话、选区、执行进度和输入框草稿。
- 小窗口不遮挡网页选择层；点选和框选仍作用于当前预览页。

### 3.3 全局状态栏

所有页面顶部保持一条状态区：

- 本地服务：未连接、连接中、已连接、连接失败。
- AI 工具：未检测、检测中、已选择、测试中、可用、不可用。
- 项目：未绑定、已绑定、预览中。
- 当前任务：空闲、选择中、执行中、待确认、已回档。

### 3.4 全局动效原则

- 状态切换使用 120-180ms 淡入或滑入，避免干扰页面预览。
- 执行进度以内嵌 AI 会话消息、工具调用状态和结果卡片呈现。
- 错误提示使用短暂震动或边框变红，不使用强烈弹窗阻断，除非涉及回档覆盖文件。
- 选择层动效要快，hover 高亮延迟不超过 50ms。

## 4. 页面一：连接检查页

### 4.1 使用时机

用户点击插件后，如果未检测到本地 companion 服务，显示此页。

### 4.2 页面内容

- 标题：连接本地服务。
- 状态说明：当前未检测到本地 companion 服务。
- 主按钮：启动或连接本地服务。
- 次按钮：查看启动说明。
- 辅助信息：本地服务用于预览 HTML、保存快照、检测本机 AI CLI、调用 AI 工具和修改本地文件。

### 4.3 交互

- 点击“连接本地服务”后，插件轮询本地服务健康检查接口。
- 连接中按钮进入 loading 状态。
- 连接成功后自动进入“首次配置页”或“AI 会话工作台”。
- 连接失败后展示失败原因和重试按钮。

### 4.4 动效

- 连接中显示按钮内旋转 loading 图标。
- 连接成功状态以绿色圆点淡入，并在 500ms 后自动跳转。
- 失败状态以红色提示条从顶部滑入。

## 5. 页面二：首次配置页

### 5.1 使用时机

本地服务已连接，但用户未完成 AI 工具配置或项目绑定时显示。

### 5.2 步骤结构

```text
1. 连接本地服务
2. 检测本机 AI CLI
3. 选择 CLI / API 模式
4. 测试 AI 工具可用性
5. 绑定 HTML 项目
6. 打开预览页
```

已完成步骤显示勾选状态，当前步骤展开。

### 5.3 AI 工具配置区

#### 内容

- 当前状态卡片：
  - 标题：本机 CLI。
  - 摘要：当前选中的 CLI、版本、模型，例如 `Codex CLI · codex-cli 0.137.0 · GPT-5.5`。
- 模式切换：
  - 使用本机 CLI。
  - 使用 API · BYOK。
- 代码代理列表：
  - Open Design AMR。
  - Claude Code，展示版本和可用状态。
  - Codex CLI，展示版本和选中状态。
- 模型选择下拉框。
- 推理强度下拉框。
- 重新扫描 PATH 按钮。
- 测试是否可用按钮。
- 设置入口。
- 安全说明：本机 CLI 模式优先复用用户本机已登录的 CLI 环境；API/BYOK 模式下 Key 仅保存在本地，不写入项目、快照或日志。

#### 交互

- 进入配置区时，本地服务自动扫描 PATH，检测可用 CLI。
- 用户可以在“使用本机 CLI”和“使用 API · BYOK”之间切换。
- 选择某个 CLI 后，模型和推理强度选项随 CLI 能力更新。
- 点击“重新扫描 PATH”后重新检测 CLI、版本和登录状态。
- 点击“测试是否可用”后，本地服务发起轻量测试，例如读取版本、发起空项目分析或 dry-run。
- 测试成功后显示“可用”，并允许进入 AI 会话工作台。
- 测试失败时显示失败原因，例如未安装、PATH 未识别、未登录、模型不可用。
- API/BYOK 模式下才展示 API Key 输入框。

#### 动效

- 扫描 PATH 时，代码代理列表显示逐项检测 loading。
- 检测到可用 CLI 后，列表项从弱态变为可选态。
- 当前选中 CLI 右侧显示勾选。
- 测试中按钮显示 loading。
- 测试成功后状态点从灰色过渡为绿色。
- 测试失败后对应 CLI 行轻微震动一次，并显示错误摘要。

### 5.4 项目绑定区

#### 内容

- 拖入 HTML 文件区域。
- 选择项目目录按钮。
- 最近项目列表。
- 当前绑定摘要：项目目录、入口 HTML、预览地址。

#### 交互

- 拖入 `index.html` 后，本地服务把该文件所在目录作为项目目录候选。
- 用户确认项目目录后，本地服务启动 `localhost` 预览。
- 点击“打开预览页”后，在当前标签或新标签打开预览地址。
- 对同一项目再次打开时，插件直接恢复绑定状态。

#### 动效

- 文件拖入区域在拖拽悬停时显示虚线高亮。
- 项目识别成功后，绑定摘要从下方滑入。

## 6. 页面三：AI 会话工作台

### 6.1 使用时机

用户已完成本地服务连接、AI 工具配置和项目绑定，并打开预览页。

### 6.2 页面结构

```text
顶部项目信息区
  项目名 / 入口 HTML / 预览地址 / 当前 AI 工具

顶部功能按钮区
  选择元素 / 框选区域 / 查看对比 / 撤销 / 快照历史 / 设置

AI 会话区
  用户需求消息
  AI 分析消息
  工具调用进度
  修改结果卡片

底部输入区
  需求输入框 / 快捷意图 / 发送按钮
```

### 6.3 顶部项目信息区

#### 内容

- 项目名或目录名。
- 入口 HTML 文件。
- 当前预览地址。
- 当前 AI 工具摘要，例如 `Codex CLI · GPT-5.5 · High`。
- 本地服务连接状态。
- 右上角窗口切换图标。

#### 交互

- 点击项目名可展开项目详情。
- 点击 AI 工具摘要可打开 AI 工具配置菜单。
- 点击右上角切换图标，在右侧边栏和独立小窗口之间切换。
- 连接异常时，状态区显示重连入口。

#### 动效

- 项目信息区保持固定，不随会话滚动。
- 切换窗口形态时，面板内容使用 160ms 缩放和淡入过渡。

### 6.4 顶部功能按钮区

#### 内容

- 选择元素按钮。
- 框选区域按钮。
- 当前选区胶囊标签。
- 查看对比按钮。
- 撤销按钮。
- 快照历史按钮。
- 设置按钮。

#### 交互

- 点击“选择元素”后，网页进入点选模式。
- 点击“框选区域”后，网页进入框选模式。
- 完成选择后，当前选区以胶囊标签形式显示，例如 `已选 div.card` 或 `已框选 482 x 260`。
- 撤销按钮默认指向最近一次修改；没有可撤销记录时禁用。
- 查看对比在存在修改记录后可用。
- 设置按钮打开设置页或 AI 工具配置菜单。

#### 动效

- 进入选择模式时，功能按钮区轻微变色，提示当前模式。
- 选区确认后，选区胶囊标签以 120ms 淡入并短暂高亮。
- 禁用按钮使用低对比样式，不改变布局尺寸。

### 6.5 AI 会话区

#### 内容

- 空状态提示：选择页面目标后，描述你想调整的效果。
- 用户消息：展示用户输入的修改要求和关联选区。
- AI 回复消息：展示分析、确认范围、修改计划和结果说明。
- 工具调用进度块：
  - 创建修改前快照。
  - 采集选区上下文。
  - 调用 AI 工具。
  - 定位和修改文件。
  - 应用补丁。
  - 刷新预览。
- 修改结果卡片：
  - 修改摘要。
  - 改动文件列表。
  - 查看对比。
  - 继续微调。
  - 撤销本次修改。

#### 交互

- 用户发送需求后，消息进入会话区，底部输入框保持可见但执行中禁用提交。
- AI 工具 CLI 输出或本地服务进度以消息块形式逐步追加。
- 每个工具步骤有状态：等待中、执行中、完成、失败。
- 如果 AI 工具请求尚未应用补丁，用户可以取消任务。
- 如果补丁已应用，取消入口转为“撤销本次修改”。
- 执行失败时，在失败步骤下展示原因和可恢复操作。

#### 动效

- AI 回复使用逐段出现，不需要模拟过慢打字效果。
- 工具调用块的当前步骤使用脉冲圆点。
- 完成步骤使用勾选动画。
- 失败步骤使用红色状态点和展开式错误说明。
- 新消息追加时会话区自动滚动到底部，但用户手动上滑查看历史时不强制抢滚动。

### 6.6 底部输入区

#### 内容

- 多行输入框，占位提示：“描述你希望这个区域如何调整”。
- 当前选区摘要小标签。
- 快捷意图标签：
  - 调整间距。
  - 弱化视觉。
  - 强化重点。
  - 修改交互。
  - 优化文案。
- 发送按钮。

#### 交互

- 未选择目标时，发送按钮可用但会先提示用户选择目标；MVP 建议默认要求先选择或框选。
- 未配置可用 AI 工具时，发送按钮禁用，并显示“先选择可用 CLI”。
- 输入框为空时，发送按钮禁用。
- 点击快捷标签后，把短语插入输入框，不直接提交。
- 支持多轮追问，继续微调时沿用最近一次选区。

#### 动效

- 输入框聚焦时边框轻微高亮。
- 发送后输入区保留位置，会话区追加用户消息和工具进度块。
- 执行中发送按钮变为停止或取消图标。

## 7. 会话内修改结果卡片

### 7.1 使用时机

AI 工具修改完成，本地文件已更新，预览页面已刷新，会话区内出现修改结果卡片。

### 7.2 卡片内容

- 状态：修改完成。
- 修改摘要。
- 改动文件列表。
- 操作按钮：
  - 查看前后对比。
  - 继续微调。
  - 撤销本次修改。
  - 保存为版本。

### 7.3 交互

- 点击“查看前后对比”打开对比与回档视图。
- 点击“继续微调”保留当前选区和输入上下文，在 AI 会话区继续输入。
- 点击“撤销本次修改”弹出确认，确认后恢复修改前快照。
- 点击“保存为版本”把当前 after 状态标记为用户认可版本。

### 7.4 动效

- 修改完成后，页面预览区域可以短暂闪烁一次细边框，提示已刷新。
- 改动文件列表逐项淡入。
- 撤销确认使用轻量确认弹层，不跳出浏览器原生弹窗。

## 8. 对比与回档

### 8.1 前后对比

#### 内容

- 对比模式切换：
  - 修改前。
  - 修改后。
  - 左右对比。
- 当前快照时间。
- 改动文件列表。
- 撤销按钮。

#### 交互

- MVP 默认支持前后截图切换。
- 如果空间足够，可以支持左右截图并排。
- 点击“修改前”时，预览区域展示 before 截图或进入 before 视觉层。
- 点击“修改后”时，返回当前预览。

#### 动效

- 前后切换使用 120ms crossfade。
- 对比模式切换时保持当前滚动位置。

### 8.2 快照历史

#### 内容

- 快照列表：时间、修改要求摘要、改动文件数、状态。
- 快照详情：before/after 文件、执行结果、恢复按钮。
- 删除快照入口。

#### 交互

- 点击快照展开详情。
- 点击“恢复到此版本”需要二次确认。
- 恢复完成后刷新预览并显示恢复状态。
- 删除快照仅删除历史记录，不影响当前项目文件。

#### 动效

- 快照详情展开使用高度过渡。
- 恢复完成后当前快照行显示成功标记。

## 9. 设置页

### 9.1 AI 工具配置

- 当前使用模式：使用本机 CLI / 使用 API · BYOK。
- 本机 CLI 检测结果。
- 代码代理列表、版本和可用状态。
- 当前选中的 CLI、模型和推理强度。
- 重新扫描 PATH。
- 测试是否可用。
- API/BYOK 模式下的模型提供商、API Key、模型名称和清除 Key。

交互要求：切换 CLI 或模型后需要重新测试可用性；清除 Key 需要确认；保存配置后立即刷新全局 AI 工具状态。

### 9.2 本地服务

- 本地服务地址。
- 连接状态。
- 重新连接。
- 当前管理的项目列表。
- 日志级别。

交互要求：切换服务地址后重新检测健康状态。

### 9.3 GitHub 增强

- 连接 GitHub。
- 当前账号。
- 当前项目远程仓库。
- 发布当前版本。

MVP 中该区为增强功能，不阻塞主流程。

### 9.4 安全与日志

- API Key 本地保存说明。
- 日志脱敏说明。
- 清除本地项目映射。
- 清除快照历史。

高风险操作需要确认，并明确说明影响范围。

## 11. 网页选择层：通用规则

### 11.1 层级结构

开启选择模式后，插件在当前网页中注入选择层。选择层位于页面内容上方、插件面板下方。

```text
原网页内容
  -> 蒙版层
  -> hover 高亮层
  -> 选中高亮层
  -> 框选矩形层
  -> 浮动操作条
  -> 插件边栏或独立小窗口
```

### 11.2 蒙版效果

#### 默认选择蒙版

- 页面整体覆盖半透明暗色蒙版。
- 建议颜色：`rgba(15, 23, 42, 0.18)`。
- 蒙版不完全遮挡内容，用户仍能看清页面结构。
- 插件面板区域不被蒙版覆盖。

#### 目标挖空效果

- hover 或选中目标区域应从蒙版中“挖空”或提高亮度。
- 如果实现挖空成本较高，MVP 可用高亮边框和浅色背景覆盖替代。
- 被选中的元素或区域保持正常亮度，周围区域轻微压暗。

#### 交互规则

- 蒙版层拦截页面普通点击，防止误触原页面交互。
- Esc 退出选择模式。
- 点击蒙版空白处不选中目标，只显示提示或取消 hover。

### 11.3 浮动提示

选择模式开启后，页面顶部或光标附近显示短提示：

- 点选模式：“点击页面元素作为修改目标，Esc 退出”。
- 框选模式：“拖拽选择一个页面区域，Esc 退出”。

提示条 2 秒后弱化，但不完全消失。

## 12. 点选元素模式

### 12.1 进入状态

用户在插件面板点击“选择元素”后：

- 页面出现半透明蒙版。
- 鼠标变为选择光标。
- 插件面板显示“正在选择元素”。
- 页面顶部出现轻量提示条。

### 12.2 Hover 高亮

当鼠标移动到可选元素上：

- 目标元素外侧显示 2px 高亮边框。
- 建议颜色：`#2563eb`。
- 边框不改变页面布局，使用 fixed/absolute overlay 绘制。
- 元素四角显示 4px 小圆点或短角标。
- 浮动标签显示元素摘要，例如 `button.primary`、`div.card`、`h2`。

### 12.3 Hover 选择策略

- 优先选中鼠标下最具体、可见、面积合理的元素。
- 对很小的文本节点，映射到其父级可交互或可布局元素。
- 按住修饰键可向父级切换选择范围。
- 如果元素被插件面板遮挡，不参与选择。

### 12.4 点击选中

用户点击 hover 元素后：

- hover 边框变为选中边框。
- 选中边框建议颜色：`#16a34a`。
- 蒙版继续存在，但选中目标保持清晰。
- 目标旁显示浮动操作条：
  - 确认目标。
  - 选父级。
  - 重新选择。
  - 取消。

### 12.5 选中后的面板反馈

插件面板显示：

- 选择类型：单个元素。
- 标签名、id、class。
- 文本摘要。
- 尺寸和位置。
- 相关父级路径。
- 主要样式摘要。

用户点击“确认目标”后，退出选择层，保留选中高亮 800ms 后淡出。

### 12.6 动效

- hover 边框跟随鼠标移动，位置更新应接近实时。
- 边框变化使用 80-120ms 过渡。
- 点击选中时边框有一次轻微放大回弹。
- 退出选择模式时蒙版 120ms 淡出。

## 13. 框选区域模式

### 13.1 进入状态

用户在插件面板点击“框选区域”后：

- 页面出现半透明蒙版。
- 鼠标变为十字选择光标。
- 插件面板显示“正在框选区域”。
- 页面顶部出现框选提示。

### 13.2 拖拽框选

用户按下鼠标并拖动：

- 起点到当前点之间显示框选矩形。
- 框选矩形边框建议颜色：`#2563eb`。
- 矩形内部填充建议：`rgba(37, 99, 235, 0.10)`。
- 矩形外区域维持蒙版压暗。
- 框选尺寸实时显示在矩形右下角，例如 `482 x 260`。

### 13.3 框选完成

用户松开鼠标后：

- 框选矩形固定。
- 插件计算区域内相关 DOM 节点。
- 区域内主要元素显示浅色描边。
- 共同父节点候选以更明显的外框展示。
- 浮动操作条显示：
  - 确认区域。
  - 扩大到父容器。
  - 缩小到可见元素。
  - 重新框选。
  - 取消。

### 13.4 区域识别规则

- 优先收集与框选矩形相交面积较大的可见元素。
- 过滤 `script`、`style`、隐藏元素和插件自身节点。
- 如果多个元素属于同一布局块，识别共同父容器作为候选。
- 如果区域覆盖整块卡片列表，保留列表容器和主要子项摘要。
- 如果区域过大，提示用户缩小范围以提高修改精准度。

### 13.5 选中后的面板反馈

插件面板显示：

- 选择类型：区域。
- 区域尺寸和坐标。
- 识别出的主要元素数量。
- 共同父节点候选。
- 区域内文本摘要。
- 区域截图缩略图。
- 主要样式和布局摘要。

用户点击“确认区域”后，退出选择层，保留区域高亮 800ms 后淡出。

### 13.6 动效

- 拖拽矩形必须跟手，无明显延迟。
- 松手后区域边框从蓝色过渡为绿色，表示已确认候选。
- 主要元素描边按从外到内顺序快速淡入。
- 重新框选时旧矩形淡出，新矩形立即可绘制。

## 14. 选择层异常状态

### 14.1 无可选元素

如果鼠标所在位置没有可选元素：

- 不显示元素边框。
- 光标旁显示“未识别到可选元素”。
- 用户可移动鼠标继续选择。

### 14.2 框选范围过小

如果框选范围小于最小阈值：

- 松手后不生成区域。
- 框选矩形闪烁红色一次。
- 提示“范围过小，请重新框选”。

### 14.3 框选范围过大

如果框选范围覆盖页面主体大部分：

- 允许继续，但提示“范围较大，修改可能不够精准”。
- 用户可确认或重新框选。

### 14.4 页面滚动

- 点选模式允许滚动页面。
- 框选拖拽到视口边缘时，页面可缓慢自动滚动。
- 滚动后高亮层坐标必须重新计算。

## 15. 原型验收清单

- 插件包含连接检查、首次配置、AI 会话工作台、会话内修改结果卡片、对比回档和设置页面。
- 插件开启后默认打开右侧边栏，并可通过右上角切换图标改为独立小窗口。
- 首次配置包含本地服务连接、本机 AI CLI 检测、CLI/API 模式配置和项目绑定。
- AI 会话工作台可以进入点选元素和框选区域两种模式。
- 点选模式有蒙版、hover 高亮、选中高亮和浮动操作条。
- 框选模式有蒙版、拖拽矩形、区域高亮、主要元素描边和浮动操作条。
- 网页蒙版不会完全遮挡页面内容。
- 被选中的元素或区域在视觉上明显高于周围内容。
- 选择完成后，插件面板能展示目标摘要。
- 修改执行中的步骤进度和失败状态体现在 AI 会话区，不使用独立执行进度页。
- 修改完成后可以查看前后对比、继续微调、撤销和保存版本。
- 快照历史支持查看详情和恢复版本。
- 本机 CLI、API/BYOK 和 API Key 安全说明在设置页中明确呈现。
